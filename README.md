# tic_tac_toe

Реализация классической игры крестики-нолики с REST API, игрой с ботом(для выбора хода бота используется алгоритм минимакс), сохранением состояния поля игры с соответсвующим ID в бд.

## 🚀 Возможности

- **REST API**: архитектурный стиль обмена сообщениями (в данном проекте в формате JSONE)  
- **Игра с ботом**: бот использует алгоритм минимакс, просчитывая все варианты ходов противника  
- **Потокобезопасность**: в качестве списка игр для текущей сессии используется потокобезопасная мапа sync.Map  
- **Сохранение в бд**: после каждого хода(в том числе и бота) состояние поля сохраняется в бд для игры с соответствующим ID
- **Начать новую игру**: при нажатии на данную кнопку и вводе ID(если игра с таким ID уже существует, приложение проинформирует об этом) создастся новая игра с пустым полем
- **Выбрать игру по ID**: при нажатии на данную кнопку, приложение проверяет, есть ли такая игра в текущей сессии(это возможно при создании или получении данных игры с таким ID). Если её нет, то пытается найти данные об игре в бд. При удачном результате поиска, игра продролжается, в противном случае приложение информирует об отсутствии игры
- **Посмотреть существующие игры**: открывается дополнительное окно с возможностью просмотра существующих игр с состоянием поля и удаления 
- **Удалённое управление игрой**: также возможно удалённое управление игрой через запросы  
- **Docker**: бд(PostgreSQL) поднимается в контейнере на порту 5433 вашего устройства, приложение в свою очередь подключается к ней   
- **MakeFile**: всё взаимодействие с приложением происходит через таргеты

## 📋 Требования

- Go 1.25.0+
- Docker Engine v29.2.1+
- Docker-Compose 2.20+
- GCC (требуется для Fyne):
  - Windows: MinGW-w64 (через MSYS2) или TDM-GCC
  - Linux: build-essential (gcc)
  - macOS: Xcode Command Line Tools

## 🛠️ Установка

### 1. Клонирование репозитория
```
git clone git@github.com:Pavlo606-oss/tic_tac_toe_go.git
cd tic_tac_toe
```

### 2. Установка зависимостей
```sh
go mod download
```

## 🎮 Использование

### Запуск контейнера с бд 
```sh
make up
```
Также для первого запуска требуется:
```sh
make migrate-up
```
Для последующих запусков этого прописывать не требуется
### Запуск игры
```sh
go run cmd/client/main.go
```
### Завершения игры
После закрытия приложения требуется прописать:
```sh
make down
```
для остановки работы контейнера, для полного его удаления:
```sh
make down-clean
```
